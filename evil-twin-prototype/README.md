# 🎓 Πτυχιακή Εργασία: Υλοποίηση Evil Twin Access Point & Captive Portal
> **Φοιτητής:** Χαράλαμπος Στίκος  
> **Τμήμα Πληροφορικής** | **Περιβάλλον Υλοποίησης: Kali Linux**

---

## 📝 Περιγραφή Έργου
Η παρούσα εργασία παρουσιάζει την αρχιτεκτονική και την υλοποίηση ενός **Evil Twin Access Point**. Το σύστημα προσομοιώνει ένα ανοιχτό δίκτυο Wi-Fi, το οποίο χρησιμοποιεί έναν **Captive Portal** server (Node.js) για την αυθεντικοποίηση των χρηστών και τη διαχείριση της πρόσβασης στο διαδίκτυο μέσω κανόνων Firewall (`iptables`).

> **Σημείωση:** Το project θα δέχεται συνεχεία αλλαγές για να αντικατοπτρίζει διαφορετικά σενάρια επιθέσεων, όπως για παράδειγμα στοχευμένες επιθέσεις σε επιχειρήσεις ή καφετέριες.

---

## 🛠️ Προαπαιτούμενα Συστήματος
Για την ορθή λειτουργία σε περιβάλλον **Kali Linux**, απαιτούνται τα εξής:

1.  **WiFi Adapter:** Συμβατός με **AP Mode** (Access Point).
2.  **Πακέτα Λογισμικού:**
    ```bash
    sudo apt update && sudo apt install hostapd dnsmasq nodejs npm -y
    ```
3.  **Εξαρτήσεις Node.js:**
    ```bash
    npm install
    ```

---

## 🚀 Οδηγίες Εκτέλεσης

Η λειτουργία του συστήματος χωρίζεται σε δύο φάσεις: τη **Ρύθμιση Δικτύου** και την **Εκκίνηση Υπηρεσιών**.

### Φάση Α: Αυτοματοποιημένη Ρύθμιση Δικτύου
Εκτελέστε το script προετοιμασίας για τη ρύθμιση των IP, του NAT και του Port Forwarding:
```bash
sudo chmod +x setup_network.sh
sudo ./setup_network.sh
```

### Φάση Β: Εκκίνηση Υπηρεσιών
Ανοίξτε **3 ξεχωριστά τερματικά** και εκτελέστε τις παρακάτω εντολές με τη σειρά:

| Terminal | Υπηρεσία | Εντολή | Περιγραφή |
| :--- | :--- | :--- | :--- |
| **#1** | **Wireless AP** | `sudo hostapd hostapd.conf` | Δημιουργία του WiFi SSID "Free_WiFi" |
| **#2** | **DHCP & DNS** | `sudo dnsmasq -C dnsmasq.conf -d` | Απόδοση IP & Ανακατεύθυνση DNS |
| **#3** | **Web Server** | `sudo node server.js` | Εκκίνηση του Captive Portal (Port 3000) |

---

## 🔍 Βασικά Σημεία Ελέγχου

*   **Ανακατεύθυνση:** Το `setup_network.sh` ανακατευθύνει αυτόματα όλη την HTTP κίνηση (Port 80) στην τοπική θύρα 3000.
*   **Αυθεντικοποίηση:** Ο server `server.js` καταγράφει τα στοιχεία στο `captures.txt` και εκτελεί δυναμικά εντολές `iptables` για να επιτρέψει την πρόσβαση στο internet στον συγκεκριμένο χρήστη.
*   **Persistence:** Τα δεδομένα των χρηστών αποθηκεύονται σε πραγματικό χρόνο για περαιτέρω ανάλυση.

---

## 📂 Δομή Αρχείων
*   `server.js`: Διαχείριση backend & Firewall rules.
*   `setup_network.sh`: Παραμετροποίηση δικτυακών διεπαφών.
*   `hostapd.conf`: Ρυθμίσεις επιπέδου Link Layer (802.11).
*   `dnsmasq.conf`: Διαχείριση επιπέδου Network (IP/DNS).

---

> **Σημείωση:** Το παρόν έργο δημιουργήθηκε στα πλαίσια ακαδημαϊκής έρευνας για την ανάλυση ευπαθειών σε δίκτυα Wi-Fi.
